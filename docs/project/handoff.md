---

## Project Overview

Trace App is a web application designed to help learners improve their trace skills for Subject B of Japan's Fundamental Information Technology Engineer Examination (FE).

The application aims to visualize where learners struggle during trace exercises, identify patterns in mistakes, and support continuous improvement through data-driven analysis.

The project also serves as a portfolio demonstrating modern web application development practices, AI-assisted development workflows, and production-ready engineering processes.

---

## Product Vision

The core concept of Trace App is:

> Help learners understand **why** they make mistakes during trace problems instead of simply telling them whether an answer is correct.

The application is intended to provide insights such as:

- Frequently missed trace steps
- Weak programming concepts
- Time spent per question
- Common mistake patterns
- Learning progress over time

The long-term goal is to make Subject B practice measurable, understandable, and continuously improvable.

## Major Decisions

Major project decisions include:

- Human is Product Owner and Tech Lead.
- Codex acts as the implementation developer.
- ChatGPT supports architecture, planning, reviews, and long-term documentation.
- AI Context is intentionally minimal and implementation-focused.
- Session Handoff is temporary and must not be committed.
- Project Handoff preserves long-term project knowledge.
- Repository documentation and AI documentation have separate responsibilities.
- Development follows an Issue-driven workflow.
- Merge commits are used for repository history.

## Long-term Constraints

The project follows several long-term principles:

- Human approves all major decisions.
- AI should not invent project information.
- Long-term knowledge belongs in Project Handoff.
- Temporary implementation state belongs in AI Context.
- Product documentation should become the primary source of product knowledge over time.
- Repository documentation should remain maintainable and avoid duplication.
