# Pull Request チェックリスト

このチェックリストは、人間がAIコーディングエージェントとの共同開発でPull Request作成時の確認漏れを防ぐために使用します。

必要に応じて、AIコーディングエージェントにもこのチェックリストを参照させてください。

## ワークフロー

GitHub Issue
↓
実装
↓
動作確認
↓
Pull Request 作成
↓
レビュー依頼

## 使用タイミング

Pull Request作成前およびレビュー依頼前に確認してください。

## Pull Request作成前

- [ ] GitHub Issue を確認した
- [ ] 実装内容を確認した
- [ ] 動作確認が完了していることを確認した
- [ ] 関連するコミットのみが含まれていることを確認した
- [ ] 現在のブランチを確認した

## レビュー依頼前

- [ ] Related Issue を設定した（例: `Closes #123`）
- [ ] Summary が今回の変更内容を正しく説明している
- [ ] Changes が実際の変更内容を正しく記載している
- [ ] Verification が実施した確認内容を正しく記載している
- [ ] Out of Scope の変更が含まれていない
- [ ] Pull Request の内容をセルフレビューした
